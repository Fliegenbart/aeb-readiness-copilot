# AEB Readiness Copilot

AEB Readiness Copilot is a pitchable MVP for trade compliance and customs
operations teams. It turns messy shipment evidence into structured Evidence
Capsules, readiness checks, remediation tasks and AEB-ready payload previews.

Prototype only. Not legal, customs, sanctions or export-control advice. The MVP
uses mock AEB adapters for AEB-adjacent workflow previews and does not connect
to external vendor systems.

## Local Setup

```bash
npm install
cp .env.example .env
npm run demo:reset
npm run dev
```

Open the URL printed by Next.js, normally `http://localhost:3000`. If that port
is already in use, Next.js may choose another local port.

The demo runs without external API keys. The default extraction provider is:

```bash
EXTRACTION_PROVIDER="deterministic"
```

For a smarter-looking local demo without external calls:

```bash
EXTRACTION_PROVIDER="mock-ai"
```

`EXTRACTION_PROVIDER="openai"` is a compiling skeleton in this MVP. It is
disabled unless `OPENAI_API_KEY` is present, and no paid external AI is required
for the demo.

## Reset Demo Data

```bash
npm run demo:reset
```

This clears local demo uploads, force-resets the SQLite schema and reseeds
deterministic Evidence Capsules, documents, extracted fields, readiness checks,
remediation tasks and audit events.

## Useful Scripts

- `npm run dev`: start the local Next.js app.
- `npm run build`: build the production app.
- `npm test`: run Vitest tests.
- `npm run db:push`: push the Prisma schema to SQLite.
- `npm run db:seed`: seed demo Evidence Capsules.
- `npm run demo:reset`: reset and reseed demo data.

## Demo Script

1. Open `/pitch` for the guided five-step walkthrough.
2. Start with messy intake: invoices, packing lists, ERP export, end-use
   evidence, technical datasheet and broker email.
3. Show the Evidence Capsule with normalized fields, confidence and source
   references.
4. Open the readiness matrix and explain ready, warning and blocked workflow
   outcomes.
5. Review contradictions, missing evidence and generated remediation tasks.
6. Show an AEB-ready payload preview from the mock AEB adapter.
7. Use “Reset demo data” to rerun the same story from a clean dataset.

## Architecture

- `app`: Next.js App Router pages, API routes and server actions.
- `components`: reusable UI pieces.
- `lib/domain`: TypeScript domain types and demo data helpers.
- `lib/extraction`: deterministic, mock AI and optional OpenAI-provider
  abstraction.
- `lib/readiness`: pure readiness and contradiction rules.
- `lib/remediation`: remediation task updates and email draft generation.
- `lib/aeb`: mock AEB adapter interfaces and AEB-ready payload previews.
- `lib/documents`: local upload validation, storage and simple extraction.
- `prisma`: SQLite schema and seed data.
- `docs`: product and architecture notes.

## Limitations

- This is not a customs filing system.
- This is not a sanctions screening engine.
- This is not a tariff classification engine.
- This is not an export-control decision engine.
- PDF uploads are stored for the demo; complex OCR is intentionally out of
  scope.
- Payloads are previews only and are generated by mock AEB adapters.
- Readiness rules identify data gaps and blockers; they do not make legal,
  customs, sanctions or export-control conclusions.

## What Production AEB Connectivity Would Require

A production path would require a separately scoped implementation effort:

- validated AEB API contracts or agreed exchange formats;
- authentication, authorization and secrets management;
- customer-specific mapping, field validation and error handling;
- audit controls, retention policies and access review;
- security review, operational monitoring and incident response;
- user acceptance testing with real process owners;
- legal and compliance review of any regulated workflow decisions.
